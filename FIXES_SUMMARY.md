# 修复方案总结

## 问题1：中英文混显
**问题**：下拉选项显示"突破 (Breakout)"格式，导致中英文混合
**原因**：INITIAL_PATTERNS 使用纯中文，但某处代码添加了英文
**修复**：
- 修改 INITIAL_PATTERNS，根据语言返回不同的选项
- 移除硬编码的中英文混合格式

## 问题2：止盈价格验证错误
**问题**：做多时，即使止盈低于入场价也显示"预计盈利"
**原因**：验证逻辑存在，但错误提示显示位置不明显
**修复**：
- 在 Take Profit 输入框下方明确显示错误提示（红色）
- 当有错误时禁用"Submit Trade"按钮

## 问题3：结算按钮触发复盘
**问题**：点击"结算"会打开复盘弹窗
**原因**：handleSettleTrade 调用错误
**修复**：
- 确保"结算"按钮打开结算弹窗
- "详情"按钮打开复盘弹窗

## 问题4：违反纪律标记
**问题**：需要标记违反纪律的交易（如撤掉止损）
**修复**：
- 在结算弹窗添加"违反纪律"复选框
- 标记的交易在列表中显示⚠️图标
- AI 分析中统计违规次数

## 问题5：总资金动态变化
**问题**：总资金固定不变，应该随盈亏变化
**修复**：
- 每次结算后自动更新 totalCapital
- totalCapital = 初始资金 + 累计盈亏
